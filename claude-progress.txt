Feature: Teller sync cursor advancing past pending transactions, causing
         settled transactions to be silently skipped on future syncs

Root cause: In sync_teller_account(), after fetching transactions from

  past them. When those transactions eventually settled and posted

  returned by Teller again — silently dropped forever.

  The account balance updates and budget threshold notifications still
  fired normally (balance is a separate API call; a pre-existing
  notification triggered the user's suspicion), masking the bug.

      After:  if posted: account.teller_cursor = posted[0].id
    Cursor now only advances to the newest *posted* transaction.
    Pending transactions above it are re-fetched on each subsequent sync
    until they settle; deduplication via external_id handles re-inserts
    safely once they post.

  - Diagnosed that the user's missing 5/22-5/25 transactions were simply
    still pending at Teller (weekend settlement lag), not a data loss
