# API Design Rules

## The four-file domain pattern

Every domain has exactly four files:
- `models.py` ‚Äî SQLAlchemy ORM model(s); no business logic
- `schemas.py` ‚Äî Pydantic request/response schemas; separate `Create`, `Update`, `Response` classes
- `service.py` ‚Äî all business logic; functions accept `AsyncSession` + domain args, return domain objects
- `router.py` ‚Äî thin FastAPI routes; call service functions, return schemas, no DB access

If a router function is longer than ~15 lines, the logic belongs in the service layer, not the router.

## Router structure

```python
@router.patch("/{entity_id}", response_model=EntityResponse)
async def update_entity(
    entity_id: uuid.UUID,
    data: EntityUpdate,
    db: AsyncSession = Depends(get_db),
    current_user = Depends(get_current_user),
):
    result = await entity_service.update_entity(db, current_user.household_id, entity_id, data)
    if result is None:
        raise HTTPException(status_code=404, detail="Not found")
    return result
```

Never call `db.add()`, `db.execute()`, or `db.commit()` directly in a router.

## Schemas

Use `model_config = ConfigDict(from_attributes=True)` on all Response schemas for ORM-to-schema mapping.

Partial updates use `model_fields_set` to update only supplied fields:
```python
for field in data.model_fields_set:
    setattr(obj, field, getattr(data, field))
```

Never use `.dict()` ‚Äî use `.model_dump()` (Pydantic v2).

Update schemas should make all fields `Optional` so clients can send partial payloads.

## Response conventions

- `200 OK` ‚Äî successful GET, PATCH, PUT
- `201 Created` ‚Äî successful POST that creates a new resource
- `204 No Content` ‚Äî successful DELETE
- `404 Not Found` ‚Äî resource doesn't exist or doesn't belong to this household (do not distinguish ‚Äî returning 404 for permission denials prevents enumeration)
- `409 Conflict` ‚Äî duplicate create (unique constraint violation)
- `422 Unprocessable Entity` ‚Äî Pydantic validation failure (FastAPI handles automatically)

## Routing prefix

Each domain router is mounted in `main.py` at `/{domain-name}` (hyphenated for multi-word domains, e.g. `/grocery-lists`). Routes inside the router are relative to that prefix.

## Async patterns

All DB operations use `await`. Never call synchronous SQLAlchemy methods (`session.execute` without `await`) from an async route. All service functions must be `async def`.

Use `db.scalar_one_or_none()` when expecting a single result that may not exist. Use `db.scalars().all()` for lists.

## Migrations

When adding a new column or table, generate a migration:
```bash
cd api && alembic revision --autogenerate -m "description"
```
Always review the autogenerated migration before applying ‚Äî SQLAlchemy does not always detect renames or JSONB type changes correctly. Run `alembic upgrade head` to apply.

## JSONB fields

Read defensively ‚Äî stored data may predate a new sub-field:
```python
cadence = habit.cadence or {}
days = cadence.get("days_of_week") or []
```

Do not add new top-level columns when a JSONB sub-field is appropriate for flexible, rarely-queried data.
